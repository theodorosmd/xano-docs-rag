Region US-WEST-01

General Access Region US-WEST-01 is located in Nevada, USA. This Region is strategically positioned to provide optimal performance and low latency for customers in the western United States and surrounding areas. It's available to all CoreWeave customers, subject to capacity constraints.

Availability Zone information​

Each Region contains one or more Availability Zones (AZs).

Each CoreWeave Availablity Zone (AZ) features a default Host Prefix, which is automatically assigned to default VPCs unless a different prefix is specified when a new VPC is created.

Info

See How-To: Create a VPC for more information about VPCs and Host Prefixes.

The AZs and their default Host Prefixes in this Region are:

Availability ZoneDefault Host Prefix CIDR
US-WEST-01A10.188.192.0/18

Public and Node address ranges​

For interoperability, each CoreWeave AZ has unique IP addresses for its Public Range, NAT Egress range, and RFC 1918 Node range.

The unique address ranges for AZ US-WEST-01A are:

DescriptionAddress RangeDescription
Public Range166.19.28.0/22The public IP address range for this AZ
NAT Egress166.19.28.0/23Subset of the Public Range used for Service egress
Node Range10.188.192.0/18RFC 1918 private range used exclusively by this AZ

Private network address ranges​

CoreWeave provides RFC 1918 private IP address ranges for Pods, Load Balancers, and other Services. These private address ranges are not unique per AZ, and may overlap with other AZs. The private ranges for all AZs in Region US-WEST-01 are:

DescriptionAddress RangeDescription
Pods10.0.0.0/13RFC 1918 space dedicated to all Pods
Services10.16.0.0/22Subset of the Pod RFC 1918 space dedicated to Services
Load Balancers10.16.4.0/22Subset of the Services RFC 1918 space dedicated to Load Balancers

Bandwidth and latency tests​

Use these speedtest files and ICMP test addresses to evaluate performance in Region US-WEST-01 for your workloads.

Test DescriptionURL/Address
1GB bandwidth test filehttp://http.speedtest.us-west-01.coreweave.com/1G

ICMP Latency test addressping.speedtest.us-west-01.coreweave.com

To run speed tests, use tools such as ping and curl, or another method of your choice. For example, to measure name lookup, connect, and total transfer times with curl, use the following command:

Example
$
curl http://http.speedtest.my-region-id.coreweave.com/1G \

       --write-out '\nDNS: %{time_namelookup}s\nConnect: %{time_connect}s\nTotal: %{time_total}s\n' \

       --output /dev/null
